Game Description

An Installable ZIP

For a user to install a new game through the Console, it must be provided in a specific file structure inside a ZIP archive:

  • game-installable-name.zip
    • images/
      • screenshot-name-1.jpg
      • screenshot-name-2.png
    • game-executable.jar
    • game-settings.json

The file names for the screenshots, the ZIP archive and executable JAR can be anything. But there must be an images folder, as well as a game-settings.json file at the root.

There can be other folders and files within the archive.

There should be at least one screenshot and they can have either the JPG or PNG formats.

Describing a Game

A game in Move2Play needs to be described by a game-settings.json file sitting in the root of the game folder.

It should contain the following fields: 

{
    "name": "Runner Race",
    "UUID": "b9c5689a-16c0-46e8-ac20",
    "description": "Jump obstacles!",
    "instructions": "Affix your phone in your armlet and start exercising.",
    "minPlayers": 1, 
    "maxPlayers": 4,
    "allowNewPlayersWhileRunning": true,
    "version": "1.0",
    "categories": ["sports", "running"],
    "developerName": "Move2Play team",
    "developerContact": "about@move2play.com.br",
    "storeId": "3",
    "images": ["ss1.jpg", "ss2.jpg"],
    "coverImageIndex": 0,
    "gameLoader": {
        "type": "game-opener",
        "settings": [
            {
                "name": "fullClassName", "type": "String",
                "value": "br.move2play.DesktopLauncher"
            }
        ]
    },
    "matchDurations": [20, 60, 120, 300, 600],
    "matchDurationIndex": 0
}
